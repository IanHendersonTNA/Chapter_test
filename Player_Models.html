Player Models with Transcripts & Chapters

This page deals with methods of presenting text transcripts of audio associated with AV files displayed within HTML web pages.

Transcript Types

Full-text transcripts derived from the audio track of AV (audio-visual) files are a means of making this sound content accessible to hard-of-hearing users,- and visually-impaired users via screen-reading devices. It also presents a more interopable and,- where a text transcript can be used or accessed independent of the AV, a more portable manifestation that can then be used in many applications.

For TNA transcripts would be particularly useful in the presentation of our video collections such as UKSC or LEV, which consist generally of long duration files containing footage of court-session, or inquiry proceedings, where near continual recorded speech is the primary information component.

Two types of transcripts can be provided:

  • Static or Plain transcripts - where the transcript is provided in text form but is independent of the AV source. These could simply be provided as downloads or presented alongside the AV player
  • Interactive transcripts, - where the transcript is delivered, integrated as part of AV file playback, and the display of text is time-synched to speech in the AV source. This is enabled by means of underlying, encoded time-indexes (- a sequence of characters or encoded information designating a particular event in time). These time-indexes can potentially be used to navigate playback along the AV duration timeline - clicking a particular text 'snippet' would navigate playback to the relevant point along the AV timeline. The automated scrolling of transcript to keep the current text within visual focus is a possible feature.

Static/ Plain Transcripts

It is of course, possible to just provide a transcript in the form of a block of text, either written within the body of the HTML page itself, or otherwise sourced from content of a separate document. Presenting a 'static' or 'plain' transcript in this manner is a simple arrangement but does not carry the benefits of interactive transcripts.


Text Support for AV in HTML5 via the <track> Element

HTML5

HTML5 is the fifth and latest major HTML version, and a W3C (World Wide Web Consortium) recommendation. The specification was designed to improve HTML structure and implementation, and added native support for multimedia. It ensures greater interopability and more consistent rendering in browsers and apps across different platforms and devices. HTML5 introduced the <audio> and <video> tags, which provide a simple methods of embedding AV within compliant webpages.

The <track> Element

<track> is a component element that can be used inside HTML5 <audio> and <video> tags and is used to reference external time-synced text files (such as WebVTT format) and thereby allow interaction with AV time-based events.

Within the <track> element there is a kind="" declaration that relates to how the external source file is being applied:

  • captions - a transcription of speech but can also include details of other sound events
  • subtitles - transcription of speech in translation
  • descriptions - text descriptions of scenario and sound events
  • chapters - navigation to pre-set points along the the AV timeline
  • metadata - metadata tracks are used to carry additional data (JSON, XML etc.) based on time-indexes. An app can then 'listen' for these cue events and as they arrive extract the text from each (e.g. processing or interaction by JavaScript via a TimedTrackCueList) and use the results to perform scripted changes or actions timed to playback.

It is possible to use multiple instances of tracks within a single video element. In the example shown in the image below there are 3 instances employed:

  1. chapters for navigation
  2. captions in English
  3. subtitles in Welsh

Descriptions are intended to be accessed by assistive-technology equipment. The text describes the visual content, set by time-code references. The text can be used for speech synthesis by screen-reader to provide vision-impaired users with an audio guide to establish a scenario and to describe events in context.

Descriptions can be employed featuring the same markup formatting as captions or subtitles, and using this screen readers can convey emphasis to the user.

WebVTT files

Web Video Text Tracks (WebVTT) originally WebSRT (Web Subtitle Resource Tracks) is a W3C (World Wide Web Consortium) standard to be used in conjunction with the HTML5 <track> element, and are used as source files for the display of text to appear in time-sync with AV playback. The format employs a simple data arrangement. The functional part of WebVTT encoding is a 'WebVTT cue block' consists of the following elements, in sequence order:

  1. cue identifier - placed on the initial line (above each sequential time-index) this can be denoted simply by sequential number, or a text string can be applied (e.g. 'Radcliffe-1', 'Outdoor-scene-1') e.g. to declare a speaker's name, or denote a scene change.
  2. cue timings - each timecode 'cue' value-pair are declared in hr: min: sec, msec format: 00:00:00,000 --> 00:00:05,000, the starting point and end-point of the timecode period being separated by a textural arrow.
  3. cue payload - the associated text to appear during the specified timecode period is written directly below the timecode. HTML style text formatting markup and CSS styling can be applied to text.

Interactive Transcripts from WebVTT Files

As well as supplying the text for captions and subtitles within the confines of the media player, WebVTT files can also be referenced in order to generate interactive transcripts. An interactive transcript contains time-synched blocks of text designated by time-indexes, generally called 'cues'. These can be used to affect AV playback and control transcript text scrolling, highlighting etc. in the following manner:

  • clicking on a particular text block within the transcript would cause AV file playback to be targeted to the corresponding point along the file's timeline
  • as playback progresses and the next point along the timeline is reached that corresponds with a cue's time-index, then the corresponding block of text in the transcript, aligned with that cue is displayed

In coded implementations the current text is often brought into focus and to the viewer's attention in some visual manner e.g. the text formatting may change from normal style, the text may be highlighted, and the text automatically scrolled to retain the visibility of the current and immediately succeeding text.

In order to employ interactive transcripts as described in the following models, programmatic interaction with HTML page elements is required, as enabled through the HTML DOM (Document Object Model).

The HTML DOM Model

HTML is a 'markup language' standard for the creation of web documents, which are structured by a hierarchical arrangement of element tags. The DOM (Document Object Model) is a W3C (World Wide Web Consortium) standard that defines methods for accessing and interacting with particular document objects. HTML DOM is a programming interface that defines an object-oriented representation of the hierarchical structure of a web-page and ensures consistency of behaviour across browsers.

Interacting with elements: the DOM Element Tree

When a web page is loaded into a browser its 'layout engine' then creates a DOM of the page structure, the various elements of which form a tree of (element or node) 'objects'. These objects are then identifiable by:

  • element type (e.g. <h1>, <div>, <p> etc.),
  • or as multiple elements defined as 'classes' (e.g. <h1 class="red-header">, <div class="box1">, <p class="these-paragraphs"> )
  • or as individual items, identified by 'id.' (e.g. <h1 id="this-one-only"> etc.).

To access a video's property or associated event, you identify it via the specific 'id' or 'class' selector specific to that video element.

<video id="video">
<source src="video.mp4" type="video/mp4">
<track label="English" kind="captions" srclang="en" src="captions.vtt" default>
</video>

When extended by the use of script (JavaScript/ JQuery) and in combination with CSS styling interaction further extended functionality is possible, beyond standard playback operation.

Enabling Interactive Transcripts with JavaScript & JQuery via the DOM

JavaScript and JQuery (JQuery - MS's open-source 'dialect' library implementation of the ECMAScript standard that 'vanilla' Javascript is also implementation of) are the scripting languages used to manipulate and control HTML DOM elements, - frequently in combination with CSS (Cascading Style Sheets) in order to effect visual change or animation.

JQuery

JQuery is a JavaScript library that was designed to be simple to use, and to overcome the issue of variable interaction and behaviour between 'vanilla' JavaScript and different browser versions. The syntax is simpler and more easily understood, and with generally much less code required to perform a task than would be needed with JavaScript.

Reportedly however, with an increasing tendency for web development work to be performed with libraries such as React and Vue, and using web application frameworks such as Bootstrap and Angular, JQuery implementation may be on the decline.

The JQuery library can be instantiated from a hosted copy of the script, or simply as a call to a CDN (Content Delivery Network) by adding this script source reference in the active HTML page: <script src="https://code.jquery.com/jquery-2.1.0.js"></script>

JavaScript interfaces with HTML via the DOM, which exposes HTML elements as objects along with their attribute values and access functions. Javascript can then interact with these objects and access attribute values. 'WebIDL' (Web Interface Definition Language) specifies the interfaces that are implemented by browser by which objects are exposed to JavaScript.

Content Attributes & IDL attributes

There are different types of attribute that can be applied to HTML elements. Attributes are values that can be used to allow HTML elements to be configured or their behaviour altered.

Content Attributes

Content Attributes are those that can be written within the HTML code and applied to relevant elements e.g. width, height, color etc. (Content Attributes are always string data types).

IDL Attributes

IDL (Interface Definition Language) Attributes (also called DOM Attributes or JavaScript property) reflect the state that an element is in and can be read or set using JavaScript. This DL attribute will use (and may also alter) an underlying content attribute for the targeted HTML element. IDL attributes may be data-types other than string. Only a few can be set that allow changing the playback of the media element.

AV Playback-Related Attributes
  • seeking
  • paused
  • ended
  • defaultPlaybackRate
  • playbackRate
  • currentTime - currentTime can be set by JS to cause navigation to a designated playback point, this then triggers a timeupdate event

(There are also 'Boolean' attributes, representing the current, switchable state of an HTML element. For example the autoplay and loop attribute can be applied to the <video> element to affect default playback behaviour).

The HTMLMedia API

The 'HTMLMedia API' is a JavaScript API and part of the HTML5 standard. Implemented in browsers it allows interaction with the HTML5 media elements - <audio> and <video> elements, and it provides a programming interface 'HTMLMediaElement', implemented in the browser, which provides developer access to the control of native HTML5 AV players. The W3C.org's HTML5 Video Events and API page presents the HTML5 video element, media API, and demonstrates media events, allowing the viewer to implement changes in video playback operation, and observe the associated changes in the displayed 'Media Property' values.

Events

Events are actions or occurrences that happen to HTML elements, which generate a 'signal' to notify when an event has occurred, and then provides a means by which a resulting, automated action may be taken - JavaScript can react to the notification and potentially a coded reaction is fired.

There are two levels of events on a web-page

  • browser level events (load, resize, scroll etc.) and
  • DOM level events applied to individual element objects or classes of DOM objects: e.g. onclick, input

A DOM event relevant to AV playback could be when the user clicks the play/ pause button, that action is an event. In the case of implementing an interactive transcript, an event could be when the user clicks on a block of text in the transcript which carries an underlying time-indexed cue. When a user clicks on that particular cue an associated time value is referenced to then target playback to that point along the video's timeline.

EventListeners

Event listeners are elements of the DOM which can be added to a web-page script in order to allow the browser to register an action or interaction performed with a page element, - such as a mouse click on a button. In the example below an eventListener is added to the <video> element.

Example: When a play event occurs this then alerts a JavaScript 'interpreter' to trigger a response - a JavaScript function runs:
video.addEventListener('play', exampleFunction)

An event can be explicitly tied to a HTML element. This is known as 'binding'. In this case, if an event involving that element is detected then the eventListener directs the JavaScript interpreter to a particular function (when a function runs as a result of an event it is known as 'handler function').

eventHandlers

EventHandlers are used to handle actions when something happens to a registered DOM element, e.g. an action to be performed when a user clicks a button. A particular portion of code (a JavaScript function) is run e.g. btn.onclick = Toggle;

In the HTML5 video example below, which features a simple interactive transcript, a JavaScript 'onclick' eventHandler is registered against the cues when a user navigates along the video time-line by clicking on a particular cue. The video will them jump to the related time-index obtained from that cue's data-time value (e.g. data-time="5.0" - here measured in seconds).

An on timeupdate event handler checks the relative positioning of the transcript text and scrolls to the top as a currently active cue is reached in playback.

duration, currentTime & timeupdate IDL attributes

In order to synch the text within an interactive transcript with the playback position of the AV file's it corresponds to, there has to be an encoded identification of relative position along the timeline by which JavaScript can action a response to a positional or time-based interaction between user navigation, either by targeted point along the AV's timeline, or from cues associated with text blocks within the transcript.

  • The duration IDL attribute returns the actual duration in seconds (as a float value)
  • The currentTime IDL attribute returns current time in seconds (as a float value) and the currentTime value can be used to allow script-based control of playback or to trigger an event when a particular value is met. currentTime can be set by JavaScript to cause navigation to a particular playback point, which then triggers a timeupdate event
  • The timeupdate IDL event is triggered whenever the current position changes, either by progressive playback or from user navigation

Video Chapters

Video chaptering is the division of a video file's entire playback duration into shorter lengths, these designated to indicate distinct point of interest or significant scene changes along the timeline. It is generally a non-edited, virtual segmentation or division of the file's overall duration into shorter sections, navigation by which is made available in an AV player, chapters appearing as demarcations along the timeline, or a button-array etc.

Chapters can be annotated to provide the viewer with a short text-based synopsis, providing the user with an overview of a videos overall content. In the player model below ('Model AV Player with Chapters') the chapters are declared simply by number sequence: 1, 2 etc.

Adding Chapters to YouTube Videos

Online video platforms such as YouTube may feature the option to add chapters to uploaded video. To add chapters to a video uploaded to YouTube all that has to be done is to add time-indexes with titles in the form: 0:00 (mins: secs) into the ‘Description’ field for the video, one for each scene change or other aspect of the footage to be chaptered. This can be done within ‘YouTube Studio’, which is a feature of a 'sign-up' YouTube account. The video timeline can be navigated from the player as shown to the right of the video Description field.

  • A Play Speed control has been provided so the user can slow the playback to better determine an exact statement, or to speed playback in order to scan for a reference or to reach an upcoming point of interest

Using Pre-Built Players with Transcript Support

As an alternative to building upon the native HTML5 video player, there are a variety of ready-built embeddable, or downloadable AV players that can be used to supply transcripts on playback. Many such as JWPlayer are proprietary but there are also open source versions such as Able Player VideoJS and Kaltura.

Rather than having to download and install, VideoJS and Kaltura players can be embedded by the addition of links to 'dependencies' (external files or libraries) in the <head> of the html document. These instantiate the player, which is served from a CDN (Content Delivery Network). One benefit of this is that you don't need to consider potential updates as these will be applied directly from the CDN. Conversely, there is a limited risk that an applied change or update may disable or disrupt some functionality that has a dependency on library elements supplied from the CDN source, or possible loss of playback service in the case of server or network issues, where local caching doesn't prevent an issue arising.

Example of links required to instantiate VideoJS Player into an HTML5 page:
<link href="https://vjs.zencdn.net/7.10.2/video-js.css" rel="stylesheet" />
<script src="https://vjs.zencdn.net/7.10.2/video.min.js"></script>

Example of an embedded instance of VideoJS Player

Enhanced Presentation Possibilities

Model Animated Video Gallery

Below is a model Video Gallery for the presentation of associated videos, i.e. those pertaining to the same event, or otherwise linked by topic or theme etc. An example would be those UKSC collection files containing footage from the same court case, or recorded on the same day of sessions. These could be arranged within a gallery contained one one HTML page so the user is able to access all relevant files in one space, and without further navigation.

With the addition of the Animatic JS animation library it is possible to animate video previews upon mouse-over of video thumbnails. When one thumbnail is clicked it is loaded to the player, should the user click another thumbnail the video currently playing back will be succeeded by the newly selected video. The original video can then be seen back in the thumbnail array but the image is paused at the point where the viewer left off. Should they recommence that video it will play back from where they left off. This could be a very useful arrangement where multiple videos are to be reviewed.

It might be possible to create and populate a gallery where the selection of related videos to be included are programmaticly selected by associated metadata references.
